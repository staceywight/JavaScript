1. JavaScript Language Fundamentals

1.1 Variables

JavaScript variables are containers for storing data values.

General Syntax

  • var myFirstVariable // declares a variable
  • myFirstVariable = value // assigns a value to the variable
  • OR........
  • var myFirstVariable = value // declares a variable and assigns a value all in one step

Methods

All JavaScript variables must be identified with unique names.
These unique names are called identifiers.
Identifiers can be short names (like x and y) or more descriptive names (age, sum, totalVolume).
The general rules for constructing names for variables (unique identifiers) are:

  • Names can contain letters, digits, underscores, and dollar signs.
  • Names must begin with a letter
  • Names are case sensitive (y and Y are different variables)
  • Reserved words (like JavaScript keywords) cannot be used as names

var x; // declares a variable
var y = 6; // assigns a value to the variable
var myName = "John Doe"; // declares a variable and assigns a value all in one step

Properties

  • myVar1; undefined data type
  • var myVar2 = "Welcome"; string data type
  • var myVar3 = 5; number data type
  • var myVar4 = "5"; string data type
  • var myVar5 = true; boolean data type
  • var myVar6 = null; object data type

1.2 Operators

Can be used to manipulate data values.

Operator = is a symbol used to act upon an item, an expression or a variable
Operand = is a variable that operators act upon

General Syntax

var myVariable;
myVariable = 1 + 1 * 2; // this variable holds the value of 3

Methods

y=4; x=y+2; => x=6 // Arithmetic

y=3 ; x=y => x=3 // Assignment

x=3; x==6 is false; x==3 is true // Comparison

x = 4 ; y = 3 // Logical
(x < 5 && y < 5) is true
(x < 5 && y > 10) is false

x = 4 ; y = 3 // Conditional
result = (x == y) ? c : d => result = d

Properties

There are different types of operators:

  1. arithmetic operators - are used to perform arithmetic between variables and values.
  2. assignment operators - are used to assign values to JavaScript variables
  3. comparison operators - are used in logical statements to determine equality or difference between variables or values
  4. logical operators - are used to determine the logic between variables or values
  5. conditional operators - assign a value to a variable based on some condition

1.3 Conditional Statements

General Syntax

if Statement Syntax:
if (expression) {statements}
The JavaScript evaluates the expression and automatically converts the result of the expression into a Boolean (true or false). If the condition is true, the statement is executed.

if …. else Statement Syntax:
if (expression) statement1 else statement2
The JavaScript evaluates the expression and automatically converts the result of the expression into a Boolean. If the condition is true, statement1 is executed. If the condition is false, statement2 is executed.

if …. else if …. else Statement Syntax:
if (condition1) statement1 else if (condition2) statement2 else statement3
The JavaScript evaluates the expression and automatically converts the result of the expressions into a Boolean. If condition1 is true, statement1 is executed. If condition2 is true, statement2 is executed. If neither condition1 and condition2 are true, statement3 is executed.

switch case Statement Syntax:
switch (expression) {
case value: statement
break;
case value: statement
break;
case value: statement
break;
case value: statement
break;
default: statement
}

Methods

If time is less than 10:00, create a "Good morning" greeting, if not, but time is less than 20:00, create a "Good day" greeting, otherwise a "Good evening":

if (time < 10) {
greeting = "Good morning";
} else if (time < 20) {
greeting = "Good day";
} else {
greeting = "Good evening";
}

1.4 Loops

Program loops are sets of commands that are executed repeatedly until a stopping condition has been met.

Any loop has three statements:
1. INITIAL state
2. CONDITION
3. INCREMENT

General Syntax

The for loop has the following syntax:

for (statement 1; statement 2; statement 3) {
code block to be executed
}

Statement 1 is executed before the loop (the code block) starts.
Statement 2 defines the condition for running the loop (the code block).
Statement 3 is executed each time after the loop (the code block) has been executed.

The while loop loops through a block of code as long as a specified condition is true. Syntax:

while (condition) {
code block to be executed
}

Methods

for (i = 0; i < 5; i++) {
    text += "The number is " + i + "<br>";
}

From the example above, you can read:

Statement 1 sets a variable before the loop starts (var i = 0).
Statement 2 defines the condition for the loop to run (i must be less than 5).
Statement 3 increases a value (i++) each time the code block in the loop has been executed.

In the following "while" example, the code in the loop will run, over and over again, as long as a variable (i) is less than 10:

while (i < 10) {
text += "The number is " + i;
i++;
}

1.5 Functions

A JavaScript function is a block of code designed to perform a particular task.
A JavaScript function is executed when "something" invokes it (calls it).

General Syntax

A JavaScript function is defined with the function keyword, followed by a name, followed by parentheses ().

Function names can contain letters, digits, underscores, and dollar signs (same rules as variables).

The parentheses may include parameter names separated by commas:
(parameter1, parameter2, ...)

The code to be executed, by the function, is placed inside curly brackets: {}

function name(parameter1, parameter2, parameter3) {
code to be executed
}

Function parameters are the names listed in the function definition.
Function arguments are the real values received by the function when it is invoked.
Inside the function, the arguments (the parameters) behave as local variables.

Methods

Convert Fahrenheit to Celsius (answer is 25):

function toCelsius(fahrenheit) {
return (5/9) * (fahrenheit-32);
}
document.getElementById("demo").innerHTML = toCelsius(77);

1.6 JS Objects - General Overview

JavaScript Objects are a collection of properties, methods and constructor(s) grouped together under one object name.

  • Properties are the values associated with an object.
  • Methods are the actions that can be performed on the object.
  • Constructors are used to create an object

There are three types of objects that can be used in JavaScript:

  • native objects (built-in objects) : objects supplied by JavaScript
     e.g Array, Date, String, Math etc
  • host objects: objects supplied by the browser environment
     e.g window, document, forms etc.
  • user-defined objects (custom objects): objects defined by programmer

The built-in objects have pre-defined methods and properties.
For custom objects, a developer has to define the methods and the properties of these objects.

General Syntax

Creating an object
1. Syntax 1: using keyword new and a constructor function:
e.g. myObj = new Date(2012);
// syntax to create object myObj as a Date Object using new keyword and Date(year) constructor

2. Syntax 2: using object literals syntax
e.g. var myObj = {property1: “wild”, property2: “flower” , property3: “red”};
e.g. var myArray ={“apple”, “banana”, “mango”, ”orange” };

Accessing Object Properties
Syntax: objectName.propertyName = value
e.g myObj.size=3; myObj.color=red;

Accessing Object Methods
Syntax: objectName.methodName
e.g myObj.setYear(2012); myObj.getYear();

Methods

Methods are actions that can be performed on objects. Methods are stored in properties as function definitions.

Property Property Value
firstName John

eyeColor blue
fullName function() {return this.firstName + " " + this.lastName;}

Properties

The name:values pairs (in JavaScript objects) are called properties.

var person = {firstName: "John", lastName: "Doe", age: 50, eyeColor: "blue"};

1.8 Array Object

The Array object is used to store multiple values in a single variable. Each element of the array is identified by an index.

afruits[0] = “apple”

In the example above, afruits is the array name, 0 is the array index and apple is the element value.

General Syntax

Option 1:
Create an array using the Array constructor with or without the new operator:

  • var aMyArray = new Array();
  • var aMyArray = Array();

Option 2:
If you know the number of elements that will be in the array, pass the count on the array constructor:

  • var aMyArray = new Array(4);
  • var aMyArray = Array(5);

Option 3:
Create an array by passing the element values in the array constructor:

  • var aMyArray = new Array(“apple”, 1, true, 5.6, “watermelon” )
  • var aMyArray = Array(“apple”, 1, true, 5.6, “watermelon” )

Using the Array Literal Notation:

  • var aMyArray = [“apple”, 1, true, 5.6, “watermelon” ];
  • var aMyArray = [] // creates an empty array; elements can be added later

Methods

  1. Conversion Methods: conversion of array object to string
  2. Stack Methods: methods that allows an array to behave like a LAST-IN-FIRST-OUT structure, the most recent added element is the first one removed
  3. Queue Methods: methods that allow an array to behave like a queue FIRST-IN-FIRST-OUT structure. A queue adds element to the end of the array and retrieve the first item in the array
  4. Reordering Methods
  5. Manipulation Methods

Properties

Property Description
constructor Returns the function that created the Array object's prototype
length Sets or returns the number of elements in an array
prototype Allows you to add properties and methods to an Array object

Length Property:
var aMyArray = new Array(“apple”, 1, true, 5.6, “watermelon” );
alert(aMyArray.length); // will return 5 which means that the array has 5 elements
The last element of the array is at position length-1 which means that the next available slot is at position length.

2.3 Events

An HTML event can be something the browser does, or something a user does.
HTML allows event handler attributes, with JavaScript code, to be added to HTML elements.
The function that is called in response to an event is called an event handler.

Event handlers can be used to handle, and verify, user input, user actions, and browser actions:

  • Things that should be done every time a page loads
  • Things that should be done when the page is closed
  • Action that should be performed when a user clicks a button
  • Content that should be verified when a user inputs data

General Syntax

<some-HTML-element some-event="some JavaScript">

<button onclick="displayDate()">The time is?</button>

Here is a list of some common HTML events:

Event Description
onchange An HTML element has been changed
onclick The user clicks an HTML element
onmouseover The user moves the mouse over an HTML element
onmouseout The user moves the mouse away from an HTML element
onkeydown The user pushes a keyboard key
onload The browser has finished loading the page
